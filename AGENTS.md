# Проездной — PWA для учёта поездок

## Описание
PWA-приложение для Android, которое считает выгоду/убыток от покупки проездного на общественный транспорт по сравнению с разовыми поездками.

## Технологии
- Чистый HTML/CSS/JS (без фреймворков)
- PWA: Service Worker + manifest.json
- Хранилище: LocalStorage
- Валюта: BYN (белорусский рубль)

## Структура файлов
```
ticket/
├── index.html       # Главный экран — учёт поездок
├── setup.html       # Экран настройки проездного
├── app.js           # Логика главного экрана
├── setup.js         # Логика настройки
├── style.css        # Стили (светлая и тёмная темы)
├── manifest.json    # PWA-манифест
└── sw.js            # Service Worker (офлайн)
```

## Экраны

### setup.html — Настройка проездного
- Дата начала
- Тип: Общий / 🚌 Наземный / 🚇 Метро
- Срок: 10 / 20 / 30 дней
- Стоимость проездного (BYN)
- Цена за поездку (BYN)
- Сохраняет дефолтные значения для следующего раза

### index.html — Учёт поездок
- Инфо: тип проездного и дата окончания
- Прогресс-бар по дням
- Поездок за сегодня + кнопка ✖️ отмены
- Кнопки 🚌 Наземный + и 🚇 Метро +
- Итого: общее число + разбивка по типам
- Расчёт: стоимость проездного vs по розничным → экономия/переплата

## Хранение данных (LocalStorage)

### `transit_pass` — текущий проездной
```json
{
  "pass": { "startDate": "2026-06-20", "duration": 30, "price": 2500, "type": "universal", "active": true },
  "trips": { "2026-06-20": { "ground": 2, "metro": 1 } },
  "log": [{ "date": "2026-06-20", "type": "ground", "ts": 1718899200000 }],
  "config": { "pricePerTrip": 46 }
}
```
- `trips` для `universal`: объект `{ ground: N, metro: M }`
- `trips` для `ground`/`metro`: просто число

### `transit_pass_defaults` — сохранённые значения формы
```json
{ "price": 2500, "tripPrice": 46, "type": "universal", "duration": 20 }
```

### `transit_theme` — тема (`"light"` или `"dark"`)

## Ключевые особенности
- Типы проездных: `universal` (общий), `ground` (наземный), `metro`
- При общем проездном — отдельный учёт наземных и метро поездок
- Лог кликов (`data.log`) — каждый +1 записывается, кнопка ✖️ отменяет последний
- Дефолтные значения формы сохраняются между настройками
- Тёмная/светлая тема с переключателем 🌙/☀️
- Офлайн-работа через Service Worker
